kafka安装、集成spring

1、安装Zookeeper

下载地址：https://zookeeper.apache.org/releases.html

下载后，放入某个目录下，例如E:\Lab\kafka目录。关于zookeeper和kafka的目录，其路径最好不要出现空格。否则运行脚本时会出现问题。

配置zookeeper：

进入zookeeper相关配置文件目录，本文目录是E:\Lab\kafka\zookeeper-3.4.10\conf。

将zoo_sample.cfg 重新命名为 zoo.cfg 。

打开zoo.cfg 将dataDir=/tmp/zookeeper 属性配置为dataDir=E:/Lab/kafka/zookeeper-3.4.10/data(具体路径可自定义)。

添加环境变量，添加ZOOKEEPER_HOME=E:\Lab\kafka\zookeeper-3.4.10(注意这里的分隔符)，在path中添加%ZOOKEEPER_HOME%\bin。

在zoo.cfg中修改默认的zookeeper端口(默认是2181,可以不修改)。

2、安装kafka

下载地址：http://kafka.apache.org/downloads

下载Binary downloads类型，解压到某个目录下面，本文解压在E:\Lab\kafka目录下。

进入kafka配置文件所在目录，E:\Lab\kafka\kafka_2.11-1.1.0\config，

编辑文件server.properties，找到并编辑：log.dirs=/tmp/kafka-logs为log.dirs=E:\\Lab\\kafka\\kafka_2.11-1.1.0\\kafka-logs，具体目录可自己指定。

端口可以不用配置，默认是9092，连接zookeeper的默认端口2181。

3、运行kafka

进入kafka安装目录E:\Lab\kafka\kafka_2.11-1.1.0。

按下shift+鼠标右键，选择在此处打开命令窗口，打开命令行，

输入命令.\bin\windows\kafka-server-start.bat .\config\server.properties。

正确运行情况为：

注：这个命令行不能关闭

 

有时输入启动命令后，出现异常： 找不到或无法加载主类

此时在kafka安装目录中找到，~bin\windows目录中的kafka-run-class.bat找到179行(也可能不是这一行）某一个%CLASSPATH%加上双引号。

修改前： 

set COMMAND=%JAVA% %KAFKA_HEAP_OPTS% %KAFKA_JVM_PERFORMANCE_OPTS% %KAFKA_JMX_OPTS% %KAFKA_LOG4J_OPTS% -cp %CLASSPATH% %KAFKA_OPTS% %*

修改后： 

set COMMAND=%JAVA% %KAFKA_HEAP_OPTS% %KAFKA_JVM_PERFORMANCE_OPTS% %KAFKA_JMX_OPTS% %KAFKA_LOG4J_OPTS% -cp "%CLASSPATH%" %KAFKA_OPTS% %*

 

4、创建主题

创建主题 myTest ， replicationfactor=1。

在目录E:\Lab\kafka\kafka_2.11-1.1.0\bin\windows目录下打开命令行。

输入命令：

kafka-topics.bat --create --zookeeper localhost:2181 --replication-factor 1 --partitions 1 --topic myTest

出现Created topic "myTest"字符说明创建成功。

5、创建生产者和消费者

在E:\Lab\kafka\kafka_2.11-1.1.0\bin\windows目录下打开命令行。

启动producer：kafka-console-producer.bat --broker-list localhost:9092 --topic myTest，该窗口不要关闭。

在同样目录下打开新的命令行。启动consumer：

kafka-console-consumer.bat --zookeeper localhost:2181 --topict myTest，在producer窗口输入任意内容，回车在consumer命令行窗口即可看到相应的内容。
